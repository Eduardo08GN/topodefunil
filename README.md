# Topodefunil — Cérebro de Copy Direct Response

Base de conhecimento viva (memória de longo prazo) sobre **copywriting de resposta direta**, construída a partir do material de copywriters que faturam múltiplos 7 dígitos.

> **Norte inicial:** [Jon Benson](mentores/jon-benson/) e [Stefan Georgi](mentores/stefan-georgi/).

## Para que serve

Preservar, organizar e conectar tudo que for aprendido — frameworks, princípios, swipes, templates e vocabulário — em formato Markdown, de modo que o conhecimento fique **estruturado para consulta e reaplicação** (não como anotação solta que se perde).

## Como está organizado

| Pasta | O que guarda |
|-------|-------------|
| [`00-mapa/`](00-mapa/) | Índice-mestre e mapa de conexões entre tudo |
| [`mentores/`](mentores/) | Perfil, métodos e princípios de cada copywriter |
| [`frameworks/`](frameworks/) | Estruturas replicáveis (VSL, big idea, leads, etc.) |
| [`hooks/`](hooks/) | Pilar de hooks para vídeo curto (reels/TikTok/anúncios) |
| [`principios/`](principios/) | Regras e verdades atemporais de copy |
| [`swipes/`](swipes/) | Trechos de copy real anotados (o que funciona e por quê) |
| [`recursos/`](recursos/) | Ferramentas e processos práticos (ex.: minerar swipes) |
| [`templates/`](templates/) | Esqueletos prontos pra escrever |
| [`glossario/`](glossario/) | Termos e jargão do direct response |
| [`_fontes/`](_fontes/) | Originais extraídos dos PDFs (texto integral, fonte de verdade) |
| [`_inbox/`](_inbox/) | Material bruto recém-chegado, a processar |

## Fluxo de trabalho

1. Você me manda a informação (texto, áudio transcrito, PDF, link, swipe).
2. Cai em [`_inbox/`](_inbox/) como material bruto.
3. Eu processo: destilo, estruturo no template certo e arquivo na pasta correta.
4. Atualizo o [índice-mestre](00-mapa/INDICE.md) e crio os links cruzados.

Convenções de escrita e templates: veja [`00-mapa/CONVENCOES.md`](00-mapa/CONVENCOES.md).
